EIMLIA-3M-TEU

Évaluation Intelligence Modèles LLM IA pour le Triage aux Urgences

Le projet EIMLIA-3M-TEU propose une évaluation rigoureuse de trois architectures d'IA pour le triage aux urgences. Nous utilisons le [cite_start]Process Mining[cite:203] pour modéliser les flux patients réels, permettant de générer des jumeaux numériques fidèles. L'étude compare notamment les performances des modèles de langage comme [cite_start]FlauBERT[cite:281] face aux approches plus récentes comme l'architecture [cite_start]JEPA[cite:320] proposée par Yann LeCun.

1. Modèle NLP (TRIAGEMASTER)

  • Basé sur [cite_start]Doc2Vec[cite:247]
  • Latence ultra-faible (120ms)
  • Extraction de symptômes clés
  • Vecteurs de paragraphes 200 dimensions

2. Modèle LLM (URGENTIAPARSE)

  • Architecture [cite_start]Transformer[cite:281]
  • Modèle BERT francophone
  • Fusion texte + constantes vitales
  • Explicabilité via [cite_start]SHAP[cite:304]

3. Modèle JEPA (EMERGINET)

  • Joint Embedding Predictive Architecture
  • Encodage conjoint Texte + Séries Temporelles
  • [cite_start]Prédiction de dégradation[cite:126]
  • Haute sensibilité urgences vitales

Méthodologie & Simulation

Pour valider ces modèles avant tout déploiement clinique, nous avons mis en place une simulation hybride. Celle-ci combine la [cite_start]Simulation à Événements Discrets (DES)[cite:98] pour les flux logistiques et les [cite_start]Systèmes Multi-Agents (SMA)[cite:102] pour simuler les comportements humains complexes. L'infrastructure repose sur un [cite_start]cluster GPU[cite:372] permettant de simuler 6 mois d'activité en quelques heures.

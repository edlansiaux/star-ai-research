Simulation Hybride & IA

EIMLIA-3M-TEU

Étude des Impacts Médico-Financiers, Logistiques de l'IA avec 3 Modèles Triant à l’Entrée des Urgences [cite_start]EIMLIA-3M-TEU[cite: 1]

Description du Projet

Le projet EIMLIA-3M-TEU vise à évaluer, par une simulation prospective validée rétrospectivement, les performances multidimensionnelles de trois architectures d'IA pour le triage aux urgences. L'étude combine [cite_start]Simulation à Événements Discrets[cite: 98] (DES) et [cite_start]Systèmes Multi-Agents[cite: 102] (SMA).

Objectif Principal

Identifier l'architecture optimale pour réduire de [cite_start]25% les temps d'attente[cite: 16] et de 20% les hospitalisations inappropriées.

Méthodologie

Utilisation avancée du [cite_start]Process Mining[cite: 203] via Celonis pour modéliser 127 parcours-types distincts.

Innovation

Comparaison inédite d'un [cite_start]Modèle JEPA[cite: 320] (Joint-Embedding Predictive Architecture) face aux LLM génératifs.

Données

Analyse de [cite_start]600 000 patients[cite: 13] du CHU de Lille sur la période 2018-2023.

Architectures IA Évaluées

1. Modèle NLP

  • Tech : [cite_start]Doc2Vec[cite: 247] + Réseau Neuronal Profond.
  • Force : Extraction rapide de symptômes critiques, latence très faible (120ms).
  • Limite : [cite_start]Compréhension contextuelle limitée[cite: 38] des cas complexes.

2. Modèle LLM

  • Tech : [cite_start]FlauBERT[cite: 281] Fine-tuné + XGBoost Hybride.
  • Force : Raisonnement clinique contextuel, explicabilité via [cite_start]valeurs SHAP[cite: 304].
  • Sécurité : Module anti-hallucinations basé sur le référentiel FRENCH.

3. Modèle JEPA

  • Tech : [cite_start]Joint-Embedding Predictive Architecture[cite: 126].
  • Force : Encodage conjoint texte/séries temporelles pour prédire la dégradation.
  • Perf : Sensibilité >96% sur les urgences vitales.

Infrastructure de Simulation

La validation écologique est assurée par une stack technique avancée permettant des stress-tests. L'infrastructure repose sur un [cite_start]Cluster GPU NVIDIA A100[cite: 372] orchestré par Kubernetes.
